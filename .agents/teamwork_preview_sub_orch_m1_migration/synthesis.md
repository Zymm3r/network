# Synthesis: Milestone 1 Database Migration

## Consensus
- The `training_lessons` table exists but is currently empty.
- Columns for `training_lessons` are: `id`, `course_id`, `title`, `video_url`, `lesson_order`, `created_at`.
- The migration must be idempotent to avoid creating duplicate records if run multiple times.
- The recommended pattern is `INSERT INTO public.training_lessons ... SELECT ... FROM public.training_courses tc WHERE NOT EXISTS ...`.
- We should use a standard placeholder YouTube embed (e.g. `https://www.youtube.com/embed/dQw4w9WgXcQ`).

## Resolved Conflicts
- Explorer 3 suggested including `id` generated by `gen_random_uuid()`. Explorer 2 omitted it but noted it might be needed if there is no default generator. It is safer to include `id` using `gen_random_uuid()` to prevent null value errors.
- Explorer 2 noted that the `title` column exists and should be populated. We will concatenate the course title with " - Placeholder Video Lesson".

## Synthesized SQL Strategy
```sql
INSERT INTO public.training_lessons (id, course_id, title, video_url, lesson_order)
SELECT 
    gen_random_uuid(),
    tc.id,
    tc.title || ' - Placeholder Video Lesson',
    'https://www.youtube.com/embed/dQw4w9WgXcQ',
    1
FROM public.training_courses tc
WHERE NOT EXISTS (
    SELECT 1 FROM public.training_lessons tl WHERE tl.course_id = tc.id
);
```
